# Cellier v2 — ViewerModel Planning

## Scope

Models required to implement the worked example from `cellier_controller_design.md`:
a button-triggered multiscale volume viewer using `CellierController`, a single scene,
a single `MultiscaleZarrDataStore`, and a single 3D canvas with an orbit camera.

## Discriminated unions

There are four discriminated unions in the v2 model layer. Each is defined as a
module-level `Annotated` type alias immediately after the concrete classes it covers.
The alias is then used as the field annotation on any model that holds that type.

All four follow the same pattern: a `Literal` tag field on each concrete class, and a
single `Annotated[Union[...], Field(discriminator="<tag>")]` alias that pydantic uses
for serialization and deserialization.

## Implementation guidelines

### ID fields

All v2 models use `UUID4` for `id` fields, not the v1 hex-string convention. Follow the
pattern already established in `BaseVisual` and `BaseDataStore`:

```python
from uuid import uuid4
from pydantic import UUID4, Field
from typing import Annotated
from pydantic import AfterValidator
import uuid

id: UUID4 | Annotated[str, AfterValidator(lambda x: uuid.UUID(x, version=4))] = Field(
    frozen=True, default_factory=lambda: uuid4()
)
```

---

### Discriminated unions

Use pydantic v2 discriminated unions for all polymorphic fields (cameras, camera
controllers). The pattern is a `Literal` tag field on each concrete class plus an
`Annotated` union at module level:

```python
from typing import Literal, Union
from typing_extensions import Annotated
from pydantic import Field

class OrbitCameraController(BaseCameraController):
    controller_type: Literal["orbit"] = "orbit"

class PanZoomCameraController(BaseCameraController):
    controller_type: Literal["pan_zoom"] = "pan_zoom"

CameraControllerType = Annotated[
    Union[OrbitCameraController, PanZoomCameraController],
    Field(discriminator="controller_type"),
]
```

The discriminator field name must match across all members of the union. The union type
alias (`CameraControllerType`, `CameraType`) is what gets used as the field annotation
on parent models.

---

### Numpy arrays

Camera models hold numpy arrays for `position`, `rotation`, `up_direction`, and
`frustum`. These are kept as `np.ndarray` (not plain tuples) because they are used in
geometric computations in the render layer.

Define a single reusable annotated type in `src/cellier/v2/types.py` that handles
coercion and serialization centrally. Every model that needs a numpy array field
annotates with this type — no per-class `@field_validator` or `@field_serializer`
boilerplate.

```python
# src/cellier/v2/types.py
import numpy as np
from pydantic import GetPydanticSchema
from pydantic_core import core_schema
from typing import Annotated

def _numpy_float32_schema():
    return core_schema.no_info_plain_validator_function(
        lambda v: np.asarray(v, dtype=np.float32),
        serialization=core_schema.plain_serializer_function_set_schema(
            lambda arr: arr.tolist(),
            info_arg=False,
        ),
    )

NumpyFloat32Array = Annotated[
    np.ndarray,
    GetPydanticSchema(lambda tp, handler: _numpy_float32_schema()),
]
```

Models that hold numpy fields must still set `model_config = ConfigDict(arbitrary_types_allowed=True)`.
This config flag is set once on `BaseCamera` so subclasses inherit it automatically.

Usage in camera models:

```python
from cellier.v2.types import NumpyFloat32Array

class PerspectiveCamera(BaseCamera):
    position: NumpyFloat32Array = Field(default_factory=lambda: np.zeros(3, dtype=np.float32))
    rotation: NumpyFloat32Array = Field(default_factory=lambda: np.array([0, 0, 0, 1], dtype=np.float32))
    up_direction: NumpyFloat32Array = Field(default_factory=lambda: np.zeros(3, dtype=np.float32))
    frustum: NumpyFloat32Array = Field(default_factory=lambda: np.zeros((2, 4, 3), dtype=np.float32))
```

### Signal relay via `model_post_init`

Sub-model events must be relayed to the parent so that the EventBus only needs to
subscribe to the top-level model. Wire these relays in `model_post_init`.

**Use closure captures — not `Signal.sender()`.** The v1 codebase uses
`Signal.sender()` to identify which sub-model fired. This is a psygnal thread-local
that can silently return the wrong object if dispatch ever crosses threads. In v2,
capture the sub-model reference explicitly in a closure at wiring time:

```python
# CORRECT — closure capture
def model_post_init(self, __context):
    for camera in self.cameras.values():
        camera.events.all.connect(
            lambda info, cam=camera: self._on_camera_updated(cam, info)
        )

# AVOID — thread-unsafe
def _on_camera_updated(self, info):
    camera = Signal.sender()   # do not use this pattern in v2
```

**Relay pattern for `BaseCamera`** — controller sub-model fires on parent's
`controller` signal:

```python
def model_post_init(self, __context):
    self.controller.events.all.connect(self._on_controller_updated)

def _on_controller_updated(self, info: EmissionInfo):
    self.events.controller.emit({"controller": {info.signal.name: info.args[0]}})
```

**Relay pattern for `Canvas`** — each camera in the `cameras` dict fires on the
canvas's `cameras` signal. Since `cameras` is a dict, iterate in `model_post_init`:

```python
def model_post_init(self, __context):
    for camera in self.cameras.values():
        camera.events.all.connect(
            lambda info, cam=camera: self.events.cameras.emit(
                {cam.id: {info.signal.name: info.args[0]}}
            )
        )
```

**Relay pattern for `Scene`** — `dims` and each visual relay upward:

```python
def model_post_init(self, __context):
    self.dims.events.all.connect(self._on_dims_updated)
    for visual in self.visuals:
        visual.events.all.connect(
            lambda info, v=visual: self.events.visuals.emit(
                {v.id: {info.signal.name: info.args[0]}}
            )
        )

def _on_dims_updated(self, info: EmissionInfo):
    self.events.dims.emit({info.signal.name: info.args[0]})
```

## Testing plan

All tests use `pytest`. No test classes — functions only. File I/O uses the `tmp_path`
fixture. Every test follows the same roundtrip pattern: construct the model, serialize
to JSON, deserialize, assert equality.

**Equality for models containing numpy arrays.** Plain `==` is unreliable when
`NumpyFloat32Array` fields are present because numpy's element-wise `==` returns an
array rather than a bool. All roundtrip assertions use `model_dump_json()` comparison
instead:

```python
assert original.model_dump_json() == deserialized.model_dump_json()
```

This is the most direct test of roundtrip fidelity: it exercises the serializer,
the JSON writer, the JSON reader, and the validator in a single assertion.

**Zarr fixture.** Tests involving `MultiscaleZarrDataStore` require an actual zarr
store on disk because `model_post_init` opens tensorstore handles synchronously.
A small shared fixture creates a minimal 2-level zarr v3 store in `tmp_path`:

```python
@pytest.fixture
def small_zarr_store(tmp_path):
    """Create a minimal 2-level multiscale zarr v3 store for testing."""
    import numpy as np
    import tensorstore as ts

    for name, shape in [("s0", (8, 8, 8)), ("s1", (4, 4, 4))]:
        level_path = tmp_path / name
        spec = {
            "driver": "zarr3",
            "kvstore": {"driver": "file", "path": str(level_path)},
            "metadata": {"shape": list(shape), "chunk_grid": {"name": "regular", "configuration": {"chunk_shape": [4, 4, 4]}}},
            "create": True,
            "delete_existing": True,
        }
        store = ts.open(spec).result()
        store[...].write(np.zeros(shape, dtype=np.float32)).result()

    return tmp_path
```

This fixture is defined in `tests/v2/conftest.py` so it is available to all v2 tests.

**`test_viewer_model_worked_example`**
This is the integration test, constructed to match the worked example from
`cellier_controller_design.md` as closely as possible without requiring the full
`~250³` blobs zarr. The `small_zarr_store` fixture provides a 2-level store at
`s0` (8³) and `s1` (4³). Construct the model by hand in the same way the controller
would internally:

```python
# Data layer
store = MultiscaleZarrDataStore(
    zarr_path=str(small_zarr_store),
    scale_names=["s0", "s1"],
    name="test_volume",
)
data = DataManager(stores={store.id: store})

# Visual
appearance = ImageAppearance(
    color_map="viridis",
    clim=(0.0, 1.0),
    lod_bias=1.0,
    force_level=None,
    frustum_cull=True,
)
visual = MultiscaleImageVisual(
    name="volume",
    data_store_id=store.id,
    downscale_factors=[1, 2],
    appearance=appearance,
)

# Scene
cs = CoordinateSystem(name="world", axis_labels=("z", "y", "x"))
dims = DimsManager(
    coordinate_system=cs,
    displayed_axes=(0, 1, 2),
    slice_indices=(),
)
camera = PerspectiveCamera(
    fov=70.0,
    near_clipping_plane=1.0,
    far_clipping_plane=8000.0,
    controller=OrbitCameraController(enabled=True),
)
canvas = Canvas(cameras={"3d": camera})
scene = Scene(
    name="main",
    dims=dims,
    visuals=[visual],
    canvases={canvas.id: canvas},
)

viewer = ViewerModel(data=data, scenes={scene.id: scene})
```

Serialize via `viewer.to_file(tmp_path / "session.json")`, deserialize via
`ViewerModel.from_file(tmp_path / "session.json")`. Assert `model_dump_json()`
equality on the full viewer. Then spot-check specific fields on the deserialized
model to confirm structural integrity:

- `len(deserialized.scenes) == 1`
- The single scene has `len(scene.visuals) == 1` and `len(scene.canvases) == 1`
- `visual.appearance.frustum_cull is True`
- `visual.appearance.force_level is None`
- `"3d" in canvas.cameras`
- `canvas.cameras["3d"].controller.controller_type == "orbit"`
- `deserialized.data.stores[store.id].n_levels == 2`
